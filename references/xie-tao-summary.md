# 谢涛教授演讲总结

> 文档：《中国电子智能体信息技术-20260701-谢涛》PDF 提取

## 一、核心战略逻辑

### 问题意识
- PC 时代：Wintel（x86）— 中国是跟随者
- 移动时代：AA（ARM）— 中国是使用者
- AI 时代：？— 不能再重蹈覆辙

### 战略选择：C 体系（"5G 模式"）
- A 体系（高铁模式）：兼容 x86/ARM → "离了你，我不行"
- B 体系（北斗模式）：全自主封闭 → "离了你，我勉强行"
- **C 体系（5G 模式）**：全开放共建 → "你中有我、我中有你"

### 时间窗口
> "未来 2-3 年是智能体技术体系和产业格局形成的关键窗口期"

## 二、灵玑OS 架构

### 四大任务
| 任务 | 名称 | 简述 |
|------|------|------|
| 任务一 | 能力引擎 | 将存量软件能力沉淀为 Skill |
| 任务二 | 记忆引擎 | 统一记忆底座 |
| 任务三 | 安全授权协议 | 智能体互联标准 |
| 任务四 | 安全防护平台 | 端云一体的可信沙箱 |

### 技术栈
- 安全内核：星绽（Rust 语言，"框内核"架构）
- 算力底座：TileLang（算子语言）+ xLLM（推理框架）
- 硬件生态：RISC-V + AI OS

## 三、与本项目的关联

### Skill 在谢教授体系中的定义
PPT 中提到的 "Skill" 不是 OpenClaw 的 SKILL.md，而是**能力引擎将存量软件封装成的可调用能力单元**。它更接近 Plugin 的定位——一个有代码逻辑的、可调用的功能模块。

### 对 create-claw-app 的启示
1. **Skill 生态需要大量开发者参与** — 能力引擎只是工具，真正的 Skill 需要人写
2. **降低开发门槛是生态繁荣的前提** — 脚手架工具是基础设施
3. **Plugin 是灵玑OS 能力引擎的天然对接形式** — 未来灵玑OS 开源的 Skill 大概率就是 Plugin
4. **现在做的工具，未来可能成为灵玑OS 的官方脚手架**

## 四、关键数据

- Token 需求：2026→2030 年每月 Token 使用量增幅约 24 倍
- 算力用电：中国算力中心用电量增幅约 4.7 倍
- 推理算力：至 2030 年推理算力占比 > 70%
- 合作伙伴：小米（人车家）、中兴（企业）、中科创达（车载）、京东（算力）
